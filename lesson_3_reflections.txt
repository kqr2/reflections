When would you want to use a remote repository rather than keeping all your work
local?
 - When I want to share the repository with other people or backup my local copy.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
 - You may be in the middle of a change.  

Describe the differences between forks, clones, and branches. When would you use one instead of another?
 - Fork is basically cloning from an existing repo on github.  Cloning means copying a repo to your local directory.  Branch creates another work space off of an existing local branch.

What is the benefit of having a copy of the last known state of the remote stored locally?
 - You can keep track of how your changes differ from the last known state of the remote

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?
 - you would have to use email in order to issue a "pull request" manually
